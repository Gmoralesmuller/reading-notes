# Git & Github

## 1. ¿Qué es el control de versiones?

El control de versiones es un sistema que registra los cambios realizados en un conjunto de archivos a lo largo del tiempo, de modo que permite recuperar versiones específicas después. También, ayuda a rastrear quién hizo cambios, qué cambios se hicieron y cuándo se hicieron.

## 2. ¿Qué es “clone” en Git?

Es un comando que se utiliza para crear una copia local de un repositorio remoto. Esto significa que descargas todos los archivos y la historia de cambios del repositorio remoto a tu computadora.

## 3. ¿Cuál es el comando para rastrear y preparar archivos?

El comando para rastrear y preparar archivos en Git es "git add"; el cual, se utiliza para agregar archivos nuevos o modificados en el **staging area** de Git, lo que significa que están listos para ser incluidos en la próxima confirmación (commit).

## 4. ¿Cuál es el comando para tomar una instantánea de los archivos modificados?

El comando para tomar una instantánea de los archivos modificados en Git es *git commit*. Después de agregar archivos al área de preparación con *git add*, puedes usar este comando para crear una confirmación **commit** con esos cambios. Cada confirmación registra un conjunto de cambios y se acompaña de un mensaje que describe esos cambios.

## 5. ¿Cuál es el comando para enviar los archivos modificados a Github?

El comando "git push" sirve para enviar tus cambios al repositorio remoto en GitHub.
